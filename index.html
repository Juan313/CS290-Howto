GOOGLE PREDICTION API

"Google Cloud Prediction API provides a RESTful API to build Machine Learning models. Prediction's cloud-based machine learning tools can help analyze your data to add various features to your applications, such as customer sentiment analysis, spam detection, recommendation systems, and more."

  1. Follow the "Before you begin" secion of this link to setup Google Cloud Platform account.

    After you sign up for the Google Cloud Platform, you will be given $300 credit to spend on the Google Cloud Platform for the next 12 months. You will need to provide credit card information and enabling billing for your project. But don't worry, they will not charge you without your consent.

  2. Once you sign up, goto Google Cloud Platform Console to create your project.

    Then, click the + sign to create a project. You need to specify a project name, the system will generate a project ID for you. Please remember the project ID because you will need it for the API calls.

  3. Creating a bucket and upload your training data.

    One thing to be careful of is that the bucket name must be globally unique.

    Once the bucket is created, you can begin uploading the training data

    To train your model, you will need to specify the storageDataLocation variable which in my case will be handwritten_digit/output.txt. Nevermind the output.txt name, you probably want to give a more sensible name to your training data than mine. Since for my demo handwritten digit recognition program, I need to do some data manipulation to meet the csv format, it's actually the output from my C++ data cleansing program. I admit it's not the best name.

  4. Last but not the least: Authorizing request with OAuth 2.0

    "Every request your application sends to the Google Prediction API must include an authorization token. The token also identifies your application to Google."

    First, go to API Manager to enable prediction API and obtain API key and client ID

    Search for "Prediction API" in the search bar"
